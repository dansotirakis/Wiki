Spring

  • Conceitos

    • O Spring surgiu em 2003 como uma resposta à complexidade do início do Java Enterprise. Em seu núcleo, o Spring começou como uma estrutura de injeção de dependência (DI) e de programação orientada a aspectos (AOP) e evoluiu para uma estrutura de aplicativo da Web fácil de usar. Por meio de sua extensa documentação, uso generalizado e inúmeras bibliotecas, o Spring permite que os desenvolvedores criem e mantenham aplicativos de forma eficiente e fornece uma curva de aprendizado plana.
    • Spring realiza DI em tempo de execução usando reflexão. Portanto, quando um aplicativo Spring é iniciado, o caminho de classe é varrido para classes anotadas. Com base nisso, os objetos concretos são instanciados e vinculados.
    • Embora seja muito flexível e amigável para o desenvolvedor, pode tornar a inicialização lenta e consumir muito memória. Além disso, é muito difícil migrar este mecanismo para GraalVM, pois ele não suporta reflexão.
